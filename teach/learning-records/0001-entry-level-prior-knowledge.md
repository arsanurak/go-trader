# Entry-level prior knowledge established

The user is a software engineer who works with the go-trader codebase daily but has not studied quantitative trading theory. They can read and apply the code (signal dispatch, config fields, backtest commands) but cannot reason independently about whether a strategy has real edge, how to read backtest statistics, or what the regime system is doing at a conceptual level.

**Implications:** Start from first principles — expected value, variance, statistical significance. Do not assume familiarity with Sharpe ratio, Kelly criterion, or regime modelling. Anchor every concept in a go-trader component they already touch.
