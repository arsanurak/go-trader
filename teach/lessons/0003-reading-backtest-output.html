อ่าน Backtest Output: Sharpe + Drawdown + N

Sharpe ratio เพียงตัวเดียวบอกไม่ครบ กลยุทธ์ที่ดีต้องผ่านทั้งสามคำถาม: edge มีคุณภาพแค่ไหน, ความเสี่ยงสูงสุดที่ต้องแบกรับคือเท่าใด, และข้อมูลมีมากพอที่จะเชื่อได้หรือยัง?

หลังจากจบบทเรียนนี้ คุณจะสามารถ

  1. อธิบาย Max Drawdown ได้ และบอกว่าทำไมมันถึงสำคัญกว่า return รวม
  2. บอกได้ว่าต้องการการเทรดกี่ครั้งจึงจะ "เชื่อ" Sharpe ที่เห็นใน backtest ได้
  3. อ่าน Discord summary ของ go-trader และตัดสินได้ว่ากลยุทธ์ควรได้รับทุนเพิ่มหรือไม่

สามเสาหลักของ Backtest

นักเทรดที่ไม่มีประสบการณ์มักดูแค่ return รวม นักเทรดที่เรียนรู้บทที่แล้วดู Sharpe แต่นักเทรดที่อ่าน backtest จริงๆ ดูทั้งสามตัวพร้อมกัน:

📐
Sharpe Ratio

คุณภาพของ edge — return ต่อหน่วยความผันผวน ถามว่า: edge มีจริงและสม่ำเสมอแค่ไหน?

📉
Max Drawdown

การขาดทุนสูงสุดจาก peak ถึง trough ถามว่า: ในช่วงที่แย่ที่สุด คุณจะทนได้ไหม?

🔢
N trades

จำนวนการเทรดใน backtest ถามว่า: ข้อมูลมีมากพอที่ตัวเลขจะมีความหมายหรือไม่?

กฎหลัก

กลยุทธ์ผ่านก็ต่อเมื่อผ่านทั้งสาม ถ้าตัวใดตัวหนึ่งล้มเหลว — Sharpe ต่ำ, Drawdown สูงเกินไป, หรือ N น้อยเกินไป — กลยุทธ์นั้นยังไม่พร้อมสำหรับเงินจริง

Max Drawdown คืออะไร

Max Drawdown (DD) คือเปอร์เซ็นต์การลดลงสูงสุดจากจุดสูงสุดไปยังจุดต่ำสุด ก่อนที่จะมี high ใหม่ในระหว่าง backtest

ตัวอย่าง

พอร์ตเริ่มต้นที่ $10,000 → ขึ้นไปที่ $13,000 (peak) → ลงมาที่ $8,500 (trough) → กลับขึ้นไปใหม่

Max DD = (13,000 − 8,500) / 13,000 = 34.6%

ณ จุดที่แย่ที่สุด คุณต้องดูพอร์ตหายไป 34.6% ก่อนจะกลับมา ถ้าคุณไม่ทนหรือตื่นตกใจและขาย — คุณจะไม่ได้ return ที่ backtest สัญญาไว้

ทำไม Drawdown ถึงสำคัญกว่า Return

เหตุผลเชิงจิตวิทยา: drawdown ที่ลึกทำให้ผู้ดำเนินการหยุดกลยุทธ์กลางคัน ก่อนที่ edge จะแสดงผล — เป็นสาเหตุที่พบบ่อยที่สุดที่นักเทรด ได้ผลลัพธ์แย่กว่า backtest จริงๆ ไม่ใช่เพราะกลยุทธ์ผิด แต่เพราะทนดู drawdown ไม่ได้

เหตุผลเชิงคณิตศาสตร์: drawdown สูงหมายถึง Sharpe ต่ำ (ความผันผวนสูง) แต่บางกลยุทธ์มี Sharpe ที่ดูดี แต่มี drawdown สูงผิดปกติ — นั่นคือสัญญาณว่ามีความเสี่ยงที่ Sharpe ไม่ได้จับ

N trades — ข้อมูลมีพอหรือยัง?

สมมติว่ากลยุทธ์หนึ่งมี Sharpe 1.5 และ Max DD 8% ฟังดูยอดเยี่ยม — แต่ถ้า backtest นั้นมีเพียง 5 การเทรด?

5 การเทรด ชนะ 4 ครั้ง Sharpe 1.5 นั้น อาจเกิดจากโชคล้วนๆ ถ้าโยนเหรียญ 5 ครั้งและได้หัว 4 ครั้ง คุณจะบอกว่าเหรียญนั้น "มี edge" ไหม?

หลักทั่วไปด้านสถิติ

ขั้นต่ำ 30 การเทรด — เพื่อให้ค่าเฉลี่ยและ std dev มีความเสถียรตามกฎจำนวนมาก (Law of Large Numbers)

50+ การเทรดขึ้นไป — เพื่อให้ Sharpe ที่คำนวณได้มีความน่าเชื่อถือพอที่จะตัดสินใจลงทุนจริง กลยุทธ์ที่เทรดบ่อย (รายชั่วโมง รายวัน) จะถึงจุดนี้เร็วกว่ากลยุทธ์ที่เทรดน้อย

hl-squeeze_momentum ของเราตอนนี้

จากสรุป Discord ที่เห็น: #T = 3 — มีเพียง 3 การเทรดใน 1 เดือน W/L = 2.00 (ชนะ 2 แพ้ 1) นั้น ไม่สามารถสรุปได้ทางสถิติ ว่ากลยุทธ์มี edge จริง ต้องการอีกอย่างน้อย 47 การเทรดก่อนที่ตัวเลขนี้จะมีความหมาย

นั่นคือ เหตุผลสำคัญที่สุด ที่ไม่ควรเพิ่มทุนในตอนนี้ — ไม่ใช่เพราะกลยุทธ์ไม่ดี แต่เพราะข้อมูลยังน้อยเกินไปที่จะรู้

PnL% — return รวมตั้งแต่ start เทียบกับ Init capital +1.4% หมายถึงพอร์ตเพิ่มขึ้น 1.4% จากทุนตั้งต้น

DD — Max Drawdown ปัจจุบัน 20% หมายถึงในช่วงนี้ พอร์ตเคยลงจากจุดสูงสุด 20% ก่อนจะกลับขึ้น — อยู่ในช่วง "ระวัง" จากตารางด้านบน

#T — จำนวนการเทรดทั้งหมด 3 — น้อยมาก ยังไม่สามารถสรุปได้ทางสถิติ

Book Sharpe — แสดงใน footer ของสรุปแบบเต็ม (ดูได้จาก localhost:8099/dashboard → Reports) นี่คือ Sharpe แบบ running จากการเทรดจริงทั้งหมดที่ผ่านมา

ฝึกตัดสิน: Backtest Judge

ดู backtest ต่อไปนี้และตัดสินแต่ละตัว: Edge จริง (ผ่านทั้งสาม), น่าสงสัย (Sharpe ดูดีเกินจริงหรือ DD สูงผิดปกติ), หรือ ข้อมูลน้อยเกินไป (N ต่ำกว่า 30)

ทดสอบความเข้าใจ

1. กลยุทธ์มี Sharpe 1.8, Max DD 7%, แต่มีเพียง 12 การเทรด ควรตัดสินอย่างไร?

ถูก: ข้อมูลน้อยเกินไป — 12 การเทรดไม่เพียงพอ ตัวเลขอาจเป็นโชค

ไม่ว่า Sharpe และ DD จะดูดีแค่ไหน 12 การเทรดไม่เพียงพอทางสถิติ ด้วยจำนวนน้อยขนาดนี้ ผลลัพธ์ที่ดีอาจเกิดจากโชคล้วนๆ ต้องรอให้ถึง 30–50 การเทรดก่อนจะตีความตัวเลขได้

2. กลยุทธ์มี Sharpe 0.7, Max DD 12%, และ 85 การเทรด HL-squeeze_momentum ของคุณมี 3 การเทรดและ DD 20% กลยุทธ์ไหนพร้อมรับทุนเพิ่มกว่ากัน และเพราะอะไร?

ถูก: กลยุทธ์แรก — N สูงกว่าทำให้ตัวเลขน่าเชื่อถือกว่า

กลยุทธ์แรกมี 85 การเทรด — เกินเกณฑ์ขั้นต่ำ ตัวเลขน่าเชื่อถือ Sharpe 0.7 อยู่ในระดับ "ยอมรับได้" และ DD 12% อยู่ในช่วงปกติ HL-squeeze มีเพียง 3 การเทรด — ไม่สามารถสรุปอะไรได้ทางสถิติ W/L 2.00 จาก 3 การเทรดไม่มีนัยสำคัญทางสถิติแต่อย่างใด

3. backtest แสดงผล Sharpe 3.2, Max DD 3%, 200 การเทรด คุณควรทำอะไรก่อน?

ถูก: ตรวจสอบ overfitting — Sharpe 3.2 สูงผิดปกติ

กลยุทธ์ที่มี Sharpe สูงกว่า 2.0 ใน backtest ควรตั้งคำถามก่อนเสมอ Sharpe 3.2 หมายถึง return สูงมากเมื่อเทียบกับความผันผวน — ในตลาดจริงนั้นหายาก ตัวเลขนี้มักเกิดจาก overfitting: กลยุทธ์ถูก "จูน" กับข้อมูลอดีตจนเกินไป และจะทำงานแย่กว่านี้มากใน out-of-sample บทเรียนถัดไปจะกล่าวถึง overfitting โดยเฉพาะ
