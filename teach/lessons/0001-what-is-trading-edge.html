การเทรดเชิงปริมาณ บทเรียนที่ 01

Trading Edge คืออะไร?

ทุกกลยุทธ์ใน codebase นี้ถูกสร้างมาเพื่อตอบคำถามเดียว: รูปแบบนี้มี ค่าคาดหวังเป็นบวก ในการเทรดหลายครั้งหรือไม่? Edge คือคำตอบ ทั้ง pipeline — signal, backtest, regime filter, live monitoring — มีอยู่เพื่อค้นหา วัดผล และรักษา edge ไว้

หลังจากจบบทเรียนนี้ คุณจะสามารถ

  1. คำนวณค่าคาดหวังต่อการเทรดจาก win rate และขนาด payoff
  2. อธิบายว่าทำไม edge จึงมองไม่เห็นในการเทรดครั้งเดียว แต่ตรวจจับได้ทางสถิติในหลายร้อยครั้ง
  3. อธิบายว่าแต่ละขั้นตอนของ go-trader pipeline เชื่อมต่อกับการค้นหาหรือรักษา edge อย่างไร

คณิตศาสตร์ของ Edge

การเทรดมีผลลัพธ์ได้สองแบบ: ชนะ หรือ แพ้ Edge คือค่าเฉลี่ยถ่วงน้ำหนักของผลลัพธ์เหล่านั้น — จำนวนเงินที่คุณคาดว่าจะได้ในการเทรดครั้งเดียว ถ้าหากรันกลยุทธ์นั้นไปเรื่อยๆ ไม่จำกัด

E[trade] = (win rate × กำไรเฉลี่ย) − (loss rate × ขาดทุนเฉลี่ย)

ถ้า E[trade] เป็นบวก กลยุทธ์นั้นทำเงินได้ในการเทรดหลายครั้ง ถ้าเป็นศูนย์ คุณกำลังพนันด้วยค่าคาดหวังเป็นกลาง ถ้าเป็นลบ คุณจะขาดทุนอย่างแน่นอนทางคณิตศาสตร์ — ไม่มีการจัดการ position sizing หรือ risk management ใดที่จะแก้ไขค่าคาดหวังติดลบได้

สูตรนี้มีนัยสำคัญที่ทำให้คนส่วนใหญ่แปลกใจ: win rate อย่างเดียวไม่บอกอะไรคุณเลย กลยุทธ์ที่ชนะ 70% ของการเทรดก็ยังมี edge ติดลบได้ ถ้าขาดทุนมากพอ

ตัวอย่างที่คำนวณแล้ว
กลยุทธ์ Win rate กำไรเฉลี่ย ขาดทุนเฉลี่ย E[trade]
A — "ดูแข็งแกร่ง" 70% $50 $200 0.70×$50 − 0.30×$200 = −$25
B — "ดูอ่อนแอ" 40% $250 $100 0.40×$250 − 0.60×$100 = +$40

กลยุทธ์ A ชนะส่วนใหญ่แต่ก็ยังขาดทุน กลยุทธ์ B แพ้ส่วนใหญ่แต่ก็ยังทำกำไร กลยุทธ์ที่มี win rate สูงกลับมี edge ติดลบ ส่วนกลยุทธ์ที่มี win rate ต่ำกลับมี edge เป็นบวก

แล้วกลยุทธ์ใน codebase นี้ล่ะ?

กลยุทธ์จริงที่รัน live ใน go-trader สังเกตใน 340 การเทรด อาจแสดงให้เห็น: win rate 53.2%, กำไรเฉลี่ย 1.42%, ขาดทุนเฉลี่ย 1.18% (เป็น return ต่อ capital)

E[trade] = 0.532 × 1.42% − 0.468 × 1.18% = 0.755% − 0.552% = +0.203%

นั่นคือ edge เล็กน้อย — แค่ 0.20% ต่อการเทรด แต่เมื่อ compound ใน 340 การเทรด ก็ได้ผลตอบแทนที่มีนัยสำคัญ scheduler/sharpe.go และ leaderboard ทำงานสะสมสถิติเหล่านี้เพื่อให้คุณเห็นว่า live edge ยังตรงกับที่ backtest ทำนายไว้หรือไม่

ทำไม Edge จึงหาได้ยาก

ปัญหาสามประการทำให้ edge ที่แท้จริงหายากและพิสูจน์ได้ยาก

1. Variance ซ่อน edge ในช่วงสั้น

แม้มี edge เป็นบวกจริง คุณก็อาจแพ้ได้ 10 ครั้งติดต่อกัน — นั่นคือ variance ปกติ ไม่ใช่หลักฐานว่า edge หายไป ด้วย edge +0.2% ต่อการเทรดและส่วนเบี่ยงเบนมาตรฐาน ต่อการเทรดประมาณ 2% คุณต้องการการเทรดราว 400 ครั้ง จึงจะมั่นใจทางสถิติได้ว่า edge ที่สังเกตได้นั้นเป็นของจริงไม่ใช่โชค นี่คือเหตุผลที่ go-trader backtests รันการเทรดหลายร้อยครั้ง ไม่ใช่หลายสิบครั้ง

2. In-sample backtests ถูก overfit กับข้อมูลในอดีต

ถ้าคุณค้นหาพารามิเตอร์นับพันชุดแล้วเลือกชุดที่ดูดีที่สุดกับข้อมูลในอดีต คุณน่าจะค้นพบความบังเอิญ ไม่ใช่ edge พารามิเตอร์ถูก tune ให้ตรงกับช่วงเวลาประวัติศาสตร์ที่เฉพาะเจาะจง นำไปใช้ในช่วงถัดไปแล้ว "edge" ก็หายไป backtest harness ของ go-trader แก้ปัญหานี้ผ่าน walk-forward validation และ economic gates — ทุกกลยุทธ์ต้องมีกลไกที่น่าเชื่อถือ (เหตุผลว่าทำไมรูปแบบนี้ควรคงอยู่) ไม่ใช่แค่ correlation ในประวัติศาสตร์

3. Edge เสื่อมลงเมื่อสภาวะตลาดเปลี่ยน

กลยุทธ์ที่มี edge แท้จริงในตลาดขาขึ้น อาจมี edge เป็นศูนย์หรือติดลบ เมื่อตลาด sideway signal เดิมยิงในทั้งสอง regime แต่ค่าคาดหวังแตกต่างกันอย่างสิ้นเชิง นี่คือเหตุผลหลักที่ระบบ regime detection มีอยู่: codebase ติดตามว่ากลยุทธ์แต่ละตัว มี edge ที่ผ่านการ validate แล้วในสภาวะตลาดไหนบ้าง และหยุดเปิด position ใหม่เมื่อสภาวะปัจจุบันไม่ตรงกับชุดนั้น

Pipeline คือตัวกรอง ในแต่ละขั้นตอนกลยุทธ์บางตัวถูกคัดออก สิ่งที่รอดจากทั้งห้าขั้นตอนควรมี edge ที่แท้จริงและยั่งยืน — ไม่ใช่แค่ความบังเอิญในประวัติศาสตร์

ใน go-trader

circuit breaker (scheduler/circuit_breaker_alert.go) ทำงานหลังจากขาดทุนต่อเนื่องห้าครั้ง การขาดทุนต่อเนื่องเป็นสัญญาณสถิติที่แข็งแกร่งกว่า การขาดทุนแบบสุ่ม — บ่งชี้ว่า regime ของตลาดอาจเปลี่ยนไปและกลยุทธ์อาจสูญเสีย edge ในสภาวะปัจจุบัน circuit breaker หยุดการเทรดชั่วคราวและบังคับให้ผู้ดำเนินการตรวจสอบ แทนที่จะเดินหน้าต่อด้วยกลยุทธ์ที่อาจมีค่าคาดหวังติดลบอยู่แล้ว

ระบบ regime detection (scheduler/regime*.go) คือ การนำ conditional edge ไปใช้โดยตรงที่สุด config allowed_regimes ของแต่ละกลยุทธ์ระบุว่าสภาวะตลาดไหนที่แสดง edge ที่ผ่านการ validate ใน backtest เมื่อตลาด live เปลี่ยนออกนอกสภาวะเหล่านั้น กลยุทธ์จะหยุดเปิด position ใหม่ — ไม่ใช่เพราะ risk limit แต่เพราะการคำนวณค่าคาดหวังไม่ถูกต้องอีกต่อไป

Sharpe tracker (scheduler/sharpe.go) วัดว่า live edge ตรงกับ backtested edge หรือไม่ การลดลงอย่างมีนัยสำคัญของ live Sharpe คือสัญญาณเตือนล่วงหน้า ว่ามีบางอย่างเปลี่ยนแปลง — ไม่ว่าจะเป็นสภาวะตลาดที่เปลี่ยนไป หรือ edge จาก backtest ไม่เคยมีอยู่จริงตั้งแต่ต้น

ทดสอบความเข้าใจ

1. กลยุทธ์หนึ่งมี win rate 55%, กำไรเฉลี่ย $80 และขาดทุนเฉลี่ย $100 edge ต่อการเทรดคือเท่าไร?

ถูก: −$1.00 ต่อการเทรด

0.55 × $80 = $44 ค่าคาดหวังการชนะ 0.45 × $100 = $45 ค่าคาดหวังการแพ้ Edge = $44 − $45 = −$1.00 Win rate 55% ยังไม่พอ — การขาดทุนมากกว่าการชนะ 25% ซึ่งมากกว่าข้อได้เปรียบของ win rate

2. ทำไม go-trader backtests จึงรันการเทรดหลายร้อยครั้งแทนที่จะเป็นหลักสิบ?

ถูก: Variance ซ่อน edge จริงในกลุ่มตัวอย่างขนาดเล็ก

ด้วย edge +0.2% ต่อการเทรดและ volatility ต่อการเทรดทั่วไปประมาณ 2% คุณต้องการ ราว 400 การเทรดเพื่อแยกแยะ edge จริงจากโชคที่ความเชื่อมั่น 95% การเทรดหลักสิบครั้งน้อยเกินไปมาก — การโชคดีต่อเนื่องมีลักษณะเหมือน edge จริง ในระดับนั้น

3. บทบาทหลักของ circuit breaker ใน go-trader เกี่ยวกับ edge คืออะไร?

ถูก: หยุดการเทรดชั่วคราวเมื่อ edge อาจเสื่อมลง

การขาดทุนต่อเนื่องห้าครั้งเป็นสัญญาณสถิติว่าสภาวะตลาดอาจเปลี่ยนไปเกินขอบเขต ที่ backtest จำลองไว้ circuit breaker บังคับให้มีการตรวจสอบโดยมนุษย์ก่อนที่การเทรดจะกลับมา แทนที่จะเดินหน้าต่อด้วยกลยุทธ์ที่อาจมีค่าคาดหวังติดลบใน regime ปัจจุบัน
