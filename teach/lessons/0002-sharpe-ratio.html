Sharpe Ratio: วัด Edge ให้คุ้มความเสี่ยง

บทเรียนที่แล้ว เรารู้ว่า edge คือค่าคาดหวังเป็นบวกต่อการเทรด แต่ edge เท่ากันสองตัวอาจมีความเสี่ยงต่างกันอย่างสิ้นเชิง Sharpe ratio คือตัวชี้วัดที่รวมทั้ง return และ volatility เข้าด้วยกัน — เป็นตัวเลขตัวเดียวที่ backtest ทุกอันควรรายงาน

หลังจากจบบทเรียนนี้ คุณจะสามารถ

  1. คำนวณ Sharpe ratio จาก mean return และ std dev ต่อการเทรด
  2. อธิบายว่าทำไมสองกลยุทธ์ที่มี return เท่ากันอาจมีคุณภาพ edge ต่างกันมาก
  3. อ่านค่า Sharpe จาก backtest output ของ go-trader และบอกได้ว่าอยู่ในระดับใด

ปัญหาของ return อย่างเดียว

สมมติว่ามีกลยุทธ์สองตัว ทั้งคู่ทำ return +25% ในปีเดียวกัน คุณควรเลือกตัวไหน?

ถ้าไม่รู้อะไรเพิ่มเติม คำถามนั้นตอบไม่ได้ เพราะ return เท่ากัน แต่ความเสี่ยงอาจต่างกันมาก ดูกรณีนี้:

ตัวอย่าง: return เท่ากัน ความเสี่ยงต่างกัน
กลยุทธ์ Return รวม Drawdown สูงสุด เดือนที่แย่ที่สุด
A — rough +25% −38% −18%
B — smooth +25% −9% −4%

กลยุทธ์ A ต้องทนดูพอร์ตจมลง 38% ก่อนจะกลับขึ้นมา นั้นคือ drawdown ที่ทำให้ผู้ดำเนินการหยุดกลยุทธ์ก่อนกำหนด ทำให้ไม่ได้รับ return ที่ backtest สัญญาไว้

Sharpe ratio แก้ปัญหานี้โดยรวม return และ volatility เข้าเป็นตัวเลขเดียว ที่สะท้อนว่า "ได้ return มากแค่ไหนต่อหน่วยของความผันผวนที่ต้องแบกรับ"

สูตร Sharpe Ratio

สูตรที่ใช้จริงใน backtest มักละ risk-free rate ออก (เพราะกลยุทธ์ระยะสั้น เปรียบเทียบกับ risk-free ไม่ได้ความ) เหลือเพียง:

S = μ / σ

โดยที่ μ คือค่าเฉลี่ย return ต่อการเทรด และ σ คือส่วนเบี่ยงเบนมาตรฐาน (standard deviation) ของ return เหล่านั้น

ตัวอย่างที่คำนวณแล้ว
กลยุทธ์ μ ต่อการเทรด σ ต่อการเทรด Sharpe = μ/σ
A — rough +0.20% 2.00% 0.10
B — smooth +0.20% 0.50% 0.40

μ เท่ากันทุกประการ ต่างกันที่ σ เท่านั้น กลยุทธ์ B มี Sharpe สูงกว่า 4 เท่า แม้จะทำ return ต่อการเทรดเท่ากัน

ทำไม σ จึงสำคัญ?

σ สูงหมายความว่าผลการเทรดแต่ละครั้งกระจายออกกว้าง — บางครั้งชนะมาก บางครั้งแพ้มาก แม้ค่าเฉลี่ยจะเป็นบวก ความผันผวนสูงหมายถึง:

  • drawdown ลึกกว่า — ต้องทนแบกขาดทุนมากกว่าก่อนจะ recover
  • ต้องใช้ trade มากกว่าเพื่อ "พิสูจน์" ว่า edge จริง (จากบทที่แล้ว)
  • position size ที่ปลอดภัยต้องเล็กกว่า — ทำให้ return รวมต่ำลง

Sharpe ต่อปี — Annualization

Sharpe ที่คำนวณจากข้อมูลต่อการเทรดนั้นขึ้นอยู่กับความถี่ของการเทรด เพื่อให้เปรียบเทียบกันได้ระหว่างกลยุทธ์ที่เทรดความถี่ต่างกัน ให้ปรับเป็น Sharpe ต่อปีโดยคูณ √N โดยที่ N คือจำนวนการเทรดต่อปี:

S_annual = S_trade × √N_trades_per_year
เทรดวันละครั้ง
(252 ครั้งต่อปี)
S_annual = S_trade × √252
≈ S_trade × 15.87
เทรดสัปดาห์ละครั้ง
(52 ครั้งต่อปี)
S_annual = S_trade × √52
≈ S_trade × 7.21

ตัวอย่าง: กลยุทธ์รายวัน μ = 0.10%, σ = 1.20% ต่อการเทรด

S_trade = 0.10 / 1.20 = 0.083 S_annual = 0.083 × √252 ≈ 1.32

ค่า Sharpe ที่ go-trader รายงานใน backtest output มักเป็น annualized Sharpe แล้ว ดังนั้นให้ตีความตามตาราง "ช่วง Sharpe" ด้านบนได้เลย

Sharpe ใน go-trader

ใน go-trader

sharpe.go (scheduler/sharpe.go) คำนวณ Sharpe ratio แบบ running สำหรับการเทรด live ทุกครั้งที่ปิด position ค่านี้จะอัพเดท ทำให้คุณเห็นได้ว่า live edge ยังตรงกับที่ backtest ทำนายไว้หรือไม่ การลดลงอย่างรวดเร็วของ live Sharpe คือสัญญาณเตือนล่วงหน้าที่แม่นยำที่สุด ว่า regime ของตลาดอาจเปลี่ยนไป

circuit_breaker_alert.go (scheduler/circuit_breaker_alert.go) ทำงานเป็น "Sharpe detector อย่างเร่งด่วน" — แทนที่จะรอให้ Sharpe ลดลงทีละน้อยข้ามสัปดาห์ มันตรวจจับการขาดทุนต่อเนื่อง 5 ครั้ง ซึ่งเป็นสัญญาณสถิติที่บ่งว่า σ เพิ่มขึ้นหรือ μ ลดลง — ทั้งคู่ทำให้ Sharpe ลดลง ทันที

backtest output (ผล run_backtest.py) รายงาน Sharpe ในบรรทัด summary เช่น: sharpe_ratio: 1.18 — ตีความตามแถบสีที่เห็นในบทเรียนนี้ กลยุทธ์ที่ผ่านเข้า live มักต้องผ่าน Sharpe ขั้นต่ำ ที่กำหนดโดยผู้ดำเนินการก่อนเสมอ

ทดสอบความเข้าใจ

1. กลยุทธ์หนึ่งมี μ = 0.30% ต่อการเทรด และ σ = 1.50% ต่อการเทรด Sharpe ratio คือเท่าไร?

ถูก: 0.20 ต่อการเทรด (= 0.30 ÷ 1.50)

Sharpe = μ / σ = 0.30% / 1.50% = 0.20 ตัวเลข 0.30 และ 1.50 มีหน่วยเป็น % เหมือนกัน จึงหักล้างกัน เหลือเป็น ratio ไม่มีหน่วย ค่า 0.20 อยู่ในโซน "ต่ำมาก" — return ไม่คุ้มความผันผวน

2. กลยุทธ์ A มี return รวม +40% แต่ Sharpe = 0.35 กลยุทธ์ B มี return รวม +18% แต่ Sharpe = 1.40 ควรเลือกตัวไหนสำหรับกลยุทธ์ระยะยาว และเพราะอะไร?

ถูก: กลยุทธ์ B — Sharpe สูงกว่า หมายถึง drawdown เล็กกว่า

Sharpe 0.35 ของ A บอกว่า return นั้นมาพร้อมกับ volatility สูงมาก สิ่งที่เกิดขึ้นคือ drawdown ขนาดใหญ่ — ในทางปฏิบัติ drawdown ที่ลึกเกินไปทำให้ผู้ดำเนินการหยุดกลยุทธ์ก่อนถึง return นั้น Sharpe 1.40 ของ B บ่งบอกว่า return ที่ต่ำกว่าสามารถ compound ได้อย่างมั่นคงกว่ามาก

3. กลยุทธ์รายวัน (252 ครั้งต่อปี) มี S_trade = 0.07 Sharpe ต่อปีคือเท่าไร? (√252 ≈ 15.87)

ถูก: 1.11 ต่อปี (= 0.07 × √252 ≈ 0.07 × 15.87)

S_annual = S_trade × √N โดยที่ N = 252 สำหรับกลยุทธ์รายวัน 0.07 × 15.87 ≈ 1.11 — อยู่ในโซน "ดี" สิ่งสำคัญ: √N ไม่ใช่ N เอง (option C) การคูณด้วย N ตรงๆ จะทำให้ค่าสูงเกินจริงมาก
